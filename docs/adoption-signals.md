# Adoption Signals

offline-verifiable authorization and execution receipts for AI and high-consequence software actions

This file tracks external technical validation and concrete usage feedback for Decision Passport Core.

## What counts as an adoption signal

- A reproducible integration write-up using exported bundles and offline verification.
- A public issue or discussion with concrete verifier, fixture, or conformance feedback.
- A third-party test report that validates or challenges verifier outputs.
- A compatibility report from another tool that consumes `1.4-basic` bundles.

## What we are actively asking for

- Verifier contract feedback: are `verdict/code/location/reason/remediation_hint` sufficient for machine triage?
- Conformance feedback: missing adversarial fixtures or ambiguous expected outcomes.
- Integration feedback: where bundle export/verify flow creates friction in agent/tool pipelines.

## How to provide high-signal feedback

1. Include exact command sequence and environment details.
2. Attach bundle(s) or a minimal reproduction fixture.
3. State expected result and actual result.
4. If proposing a contract change, describe compatibility impact.
